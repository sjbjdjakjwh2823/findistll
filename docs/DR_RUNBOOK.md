# DR Runbook (Preciso)

## 목표
- RTO/RPO 기준에 맞춰 서비스 복구를 수행한다.

## 준비
- `SUPABASE_DB_URL` 또는 운영 DB 접속 정보 확보
- 백업 경로 확인
- 롤백 스크립트 위치 확인: `scripts/rollback.sh`

## 복구 절차
1. 장애 원인 분류 및 영향 범위 확인
2. 서비스 중지 또는 읽기 전용 모드 전환
3. 최근 백업 확인
4. 복구 실행
5. 스모크 테스트 및 핵심 API 확인
6. 장애 종료 보고

## 체크리스트
- API `/api/v1/status` 정상
- 검색/승인/감사 이벤트 로그 확인
- 데이터 무결성 점검 완료
